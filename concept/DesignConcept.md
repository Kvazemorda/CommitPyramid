# CityDeveloper — Design Concept

_Источник правды для UI-задач. Все цвета, шрифты, отступы в задачах ссылаются сюда._

_Файл предварительный — кода и ассетов ещё нет. Значения подобраны как стартовая
палитра в стилистике «упрощённая египетская изометрия / Pharaoh». Будут уточнены при
появлении первого арт-прохода (D-13)._

## Стилистика

**Жанр:** изометрический city-builder в духе Pharaoh / Caesar III, но упрощённый.
**Эпоха:** условно-древняя (Египет / Месопотамия).
**Палитра:** тёплая, охровая, песочная. Зелень — приглушённая (Нил/оазисы). Контраст
держится на тенях и силуэтах, а не на ярких цветах.
**Целевой look:** камерная диорама на рабочем столе, не отвлекает в wallpaper-режиме,
радует деталями в explore-режиме.

## Компоненты (планируемые SwiftUI)

### Inspector Popover (попап над юнитом)
- Фон: `parchment` с лёгкой текстурой папируса (image overlay 8% opacity)
- Бордер: 1 pt `inkDark` 30% opacity
- Тень: 10 pt blur, `inkDark` 25% opacity
- Заголовок: 15 pt Semibold `inkDark`
- Контент: 13 pt Regular `inkDark` 85%
- Действия (ссылка на источник): `info` underline

### Side Panel (журнал событий)
- Фон: `parchment` 95% opacity, blur background
- Ширина: 320 pt fixed
- Скролл: нативный, без видимой полосы по умолчанию

### Project Card (в списке проектов)
- Фон: `sandLight`
- Заголовок: имя проекта 13 pt Semibold
- Метрики: население / юнитов / последняя активность — 11 pt caption
- Индикатор decay: цветная полоса слева (`success` → `warning` → `danger`)

## Layout-паттерны

- **Wallpaper-режим:** только SpriteKit scene, без SwiftUI оверлеев. Полный экран.
- **Explore-режим:** SpriteKit scene + правая боковая SwiftUI-панель (collapsible).
  Поверх — попап-инспектор по клику на юнит.
- **Settings:** отдельное окно, стандартный macOS look (Form/GroupBox SwiftUI).

## Breakpoints и мобайл
Не применимо — desktop-only macOS. Адаптация под размер окна:
- минимальный размер окна explore-режима: 1024 × 640 pt;
- wallpaper-режим всегда на полный экран основного дисплея.

## Анимации (планируемые)

| Событие                         | Длительность | Easing            |
|---------------------------------|--------------|-------------------|
| Появление нового юнита          | 600 ms       | spring (mass=1)   |
| Апгрейд стадии (smooth-replace) | 800 ms       | easeInOut         |
| Decay уровня (медленное появл.) | 1500 ms      | easeOut           |
| Restore-анимация (стройка)      | 3000–5000 ms | scripted sequence |
| Переключение wallpaper↔explore  | 180 ms       | easeOut           |
