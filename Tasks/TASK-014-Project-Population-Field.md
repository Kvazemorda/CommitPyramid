### Что хотим

Добавить поле «население» в карточку проекта боковой панели (F-11). По концепту,
люди — следствие активности проекта: чем больше активных юнитов, тем больше
жителей. Сейчас `ProjectState` поля `population` не содержит — есть только
computed `CityState.population` (сумма по всему городу), что не подходит для
per-project отображения.

### Почему отложено и почему нужно разбить

Задача нетривиальна и затрагивает несколько слоёв:

1. **Модель** — новое поле `ProjectState.population` (или helper-функция).
2. **Формула** — как считать. По концепту F-10:
   > Население квартала = функция от количества юнитов и stage.
   Точная формула не зафиксирована в концепте. Нужно либо взять из TASK-011
   (`target = min(20, stage * 2 + units / 4)`), либо договориться об
   отдельной формуле для отображаемого «населения» (например, реальное
   количество жителей на сцене из CitizenManager).
3. **Источник истины** — где считается: при applyTaskCompleted? в
   CitizenManager.tick? при чтении из UI? Это влияет на консистентность.
4. **UI** — карточка проекта (TASK-007 → ProjectCard) получает дополнительное
   поле; ожидается лёгкая правка.

**Зависимости от других задач:**
- TASK-011 (жители) — даёт реальный счётчик; без него «население» = умозрительная цифра.
- TASK-007 (Inspector Panel) — карточка проекта, куда добавляется поле.

### Предлагаемое разбиение на следующий цикл

- **TASK-014a:** Модель + формула. Добавить `ProjectState.population: Int` (или
  computed property через CityEngine helper). Зафиксировать формулу. Replay-safe.
- **TASK-014b:** Интеграция в карточку проекта (ProjectCard из TASK-007).
  Маленький UI-патч.

После /sync следующего цикла (через /pm) — оформить эти подтаски с полными
AC, edge cases, dependencies. До тех пор задача `deferred`.
