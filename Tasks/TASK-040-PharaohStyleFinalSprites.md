### Что хотим

После решения пользователя выложить проект на GitHub скоуп задачи меняется. **Не делаем сами 50 финальных PNG**. Вместо этого:

1. Пишем подробные **AI-промпты** для каждого нужного спрайта (50 юнитов + 7 биомов + decay-варианты) — складываем в `docs/asset-prompts.md`.
2. Подтверждаем, что **fallback-механизм работает**: если PNG есть в `Sources/CityDeveloper/Resources/Buildings/<name>.png` — используется, если нет — программный placeholder из TASK-032 (`makeKindBuilding`).
3. Документируем для контрибьюторов: как сгенерировать PNG по промпту, куда положить, как назвать, какой формат. Этот документ — `docs/CONTRIBUTING-ASSETS.md` (на английском, см. F-21).
4. Procedural «пластиковые» визуалы остаются дефолтом — играть можно без единого PNG.

### Почему так

- Пользователь хочет выложить проект open-source. Идея — «коммиты растят город».
- Контрибьюторы-художники возьмут промпты, прогонят через AI-генератор (Midjourney/DALL·E/SDXL), пришлют PR с PNG. Каждый PR добавляет визуальную fidelity к проекту.
- Pro-художник (~$300–800) и ручная AI-генерация одной командой требуют большого focus от автора. Контрибьюторская модель распределяет это.

### Пользовательский сценарий

1. Контрибьютор клонирует репозиторий, читает `docs/CONTRIBUTING-ASSETS.md`.
2. Открывает `docs/asset-prompts.md`, выбирает нужный спрайт (например, Pyramid).
3. Копирует промпт в Midjourney / DALL·E / SDXL → получает PNG.
4. Конвертирует в формат (RGBA, прозрачный фон, размер по `sprite-generation-rules.md` § 1).
5. Кладёт в `Sources/CityDeveloper/Resources/Buildings/pyramid.png`.
6. Открывает PR. После merge — `swift build` подхватит ассет автоматически (PNG-first fallback из TASK-032).

### Acceptance criteria

- [ ] Создан `docs/asset-prompts.md` (английский). Структура:
  - **Header:** назначение, формат, стиль.
  - **Buildings (50):** на каждый юнит — блок с `kind` (rawValue из UnitKind), имя файла (`<rawValue>.png`), размер, **prompt** 3–5 предложений английского текста (укажи stage, категорию, изометрию, палитру), 1–2 negative-промпта.
  - **Biomes (7):** аналогично для текстур биомов.
  - **Decay variants:** для каждого здания stage 5 — декей-вариация.
- [ ] Создан `docs/CONTRIBUTING-ASSETS.md` (английский) с пошаговой инструкцией: где промпт, какие AI-генераторы, технические требования к PNG, куда положить, как протестировать локально, как открыть PR.
- [ ] **PNG-first fallback подтверждён:** в `UnitSprites.makeKindBuilding(unit:stage:)` (TASK-032) механизм уже работает. Здесь — regression-проверка одним smoke: тестовый `shack.png` (уже добавлен) используется, не placeholder.
- [ ] `concept/SpriteGenerationRules.md` переведён в `docs/sprite-generation-rules.md` (английский). Старый файл удалить.
- [ ] В `README.md` (см. F-21) добавлена ссылка на `docs/asset-prompts.md` в секцию «How to contribute».

### Что НЕ делаем

- **НЕ генерируем сами 50 финальных PNG** (это работа контрибьюторов или будущая задача).
- НЕ заказываем художника.
- НЕ делаем automatic AI-генерацию (бэкэнд / CI с Midjourney API).
- НЕ удаляем placeholder-спрайты — они остаются дефолтом.

### Edge cases

- [ ] PNG неверного размера / не RGBA — пользователь увидит искажение. Документировать как «проверьте до commit».
- [ ] Дублирующиеся имена файлов в PR — резолвить через PR review.
- [ ] Контрибьютор использует UnitKind, которого нет в enum — отклонить PR с ссылкой на TASK-031.
- [ ] Stage-варианты: один PNG на все stage (placeholder) vs PNG на каждый stage (5×) — оба legal, fallback подставит существующий.
